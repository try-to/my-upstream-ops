# 按启用 API Key 过滤渠道倍率分组设计

## 目标

渠道卡片只展示当前渠道中至少被一个正常启用 API Key 使用的倍率分组。未被启用 Key 使用的分组不展示，避免把上游返回的全部分组误认为当前账号正在使用。

## 规则

- 只统计 `status=active` 的 API Key。
- API Key 的远端分组 ID与倍率快照的远端分组 ID相同时视为匹配。
- 没有远端分组 ID时，使用规范化后的分组名称匹配 NewAPI 等兼容接口。
- API Key 列表必须完整分页读取后才执行过滤。
- API Key 查询失败时返回未过滤的倍率列表，避免网络异常误隐藏所有分组。
- 没有任何启用 API Key 时返回空倍率列表；现有前端自然隐藏分组区域。
- 不向前端返回 API Key 内容，只返回过滤后的倍率快照。

## 实现方案

`GET /api/channels/:id/rates` 在读取本地倍率快照和策略后，通过现有渠道服务复用认证会话分页读取 API Key。后端建立“远端分组 ID”和“规范化分组名称”两组集合，过滤倍率快照，并继续保留现有倍率策略字段与自动调度状态。

分页使用连接器允许的最大页大小，依据返回的 `pages` 和 `total` 继续读取，避免只检查第一页。启用状态判断使用连接器已统一的 `active` 值，因此兼容 NewAPI 和 Sub2API 的原始状态差异。

## 错误处理

读取 Key 失败时不让 `/rates` 失败，也不执行过滤，维持当前倍率展示，避免临时认证或网络问题造成错误 UI。错误写入服务日志时只包含渠道 ID和错误类型，不记录 Key 或凭据。

## 测试

- 分页读取多页 API Key，并只收集启用 Key 的分组。
- 远端分组 ID优先匹配，名称回退匹配。
- 没有启用 Key 返回空列表。
- Key 查询失败时保留未过滤列表。
- `/rates` 继续正确合并倍率策略字段。
