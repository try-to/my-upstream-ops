# 渠道分组倍率自动控制 Sub2API 调度设计

## 目标

按照每个上游渠道分组独立配置的最大允许倍率，自动启用或禁用该分组绑定的 Sub2API 托管账号调度。

配置粒度必须是 `渠道 + 渠道分组`，不能继续使用同步分组共用阈值。功能只修改当前 UpstreamOps 项目，通过 Sub2API Admin API 更新账号 `schedulable`，不修改 Sub2API 源码和账号 `status`。

## 已确认规则

- 每个渠道分组可以独立设置最大允许倍率。
- 每个渠道分组可以设置计算比例，默认值为 `1`，支持小数。
- 判定倍率为：`当前渠道分组倍率 × 计算比例`。
- 判定倍率大于最大允许倍率时，禁用对应托管账号调度。
- 判定倍率小于或等于最大允许倍率时，重新启用对应托管账号调度。
- 未设置最大允许倍率时，不自动改变调度状态。
- 倍率读取失败、分组消失或没有当前倍率时，不改变调度状态。
- 同一渠道分组绑定多个 Sub2API 托管账号时，全部执行相同决策。
- 只处理由 UpstreamOps 管理且同步账号、同步分组、同步目标均启用的真实账号；不处理手工创建账号和 OAuth 虚拟池。

## 方案选择

使用独立的渠道分组倍率策略表，不把策略直接写入倍率快照或同步账号。

倍率快照在分组暂时消失时会被删除，不适合作为长期配置载体。同步账号配置会导致同一渠道分组重复设置，也不符合渠道分组级语义。独立策略可以在分组暂时消失时保留配置，并由多个托管账号共享。

## 数据模型

新增渠道分组倍率策略，核心字段：

```text
channel_id
group_key
remote_group_id（可空）
group_name
max_ratio
calculation_ratio（默认 1）
created_at
updated_at
```

`group_key` 在同一渠道内唯一：有远端分组 ID 时使用 ID 构造；没有 ID 时使用规范化分组名称构造。策略同时保存当前分组名称用于展示，匹配时优先使用远端 ID，名称作为兼容回退。

清空最大倍率表示关闭该渠道分组的自动启停，可以直接删除对应策略记录。原同步分组中的 `rate_auto_toggle_threshold` 和 `rate_auto_toggle_ratio` 不再参与决策，不自动迁移到渠道分组，避免把一个旧阈值错误复制给多个不同分组。旧数据库字段保留兼容，不在新 UI 中展示。

## 调度数据流

### 定时扫描

1. 调度器执行现有渠道倍率扫描并更新 `rate_snapshots`。
2. 扫描完成后调用渠道分组策略协调器。
3. 协调器读取已配置策略、当前倍率快照、同步账号和托管账号映射。
4. 通过 `source_channel_id` 和 `source_group_id/source_group_name` 找到绑定该渠道分组的同步账号。
5. 过滤未启用的同步账号、同步分组和同步目标。
6. 计算判定倍率并调用目标 Sub2API 的 `schedulable` 接口。
7. 单个账号更新失败只记录错误，继续处理其他账号。

### 手动刷新

用户手动刷新单个渠道倍率成功后，只协调该渠道的策略，使手动操作与定时扫描得到相同结果。

协调器只执行调度状态变更，不重新创建账号、密钥或目标分组，不运行模型测试，也不重新应用完整同步分组。

## 状态覆盖规则

倍率策略是已配置渠道分组的最终调度决策。只要最大倍率存在，每次成功扫描都会根据最新倍率设置目标账号调度：超过阈值设为禁用，恢复到阈值以内设为启用。

没有策略或没有有效倍率时返回 `skip`，不得为了“恢复默认值”主动启用账号。远端账号本身为 `inactive` 时仍只修改 `schedulable`，不改变 `status`。

## API

渠道倍率响应增加当前分组策略显示字段，浏览器不需要自行拼接数据库记录：

```text
GET /api/channels/:id/rates
```

每个分组可返回：

```text
max_ratio: nullable number
calculation_ratio: number
calculated_ratio: nullable number
auto_schedulable_state: unconfigured | enabled | disabled
```

新增保存接口：

```text
PUT /api/channels/:id/rate-groups/policy
```

请求包含远端分组 ID（可空）、分组名称、最大倍率（可空）和计算比例。服务端验证渠道及当前分组存在；最大倍率必须大于等于 `0`，计算比例必须大于 `0`。最大倍率为空时删除策略。

API 不接受 Sub2API 账号 ID，防止浏览器绕过托管映射直接控制任意账号。

## 前端交互

渠道卡片中的每个分组倍率标签改为可点击按钮，并明确提示“点击配置自动启停”。点击后打开轻量配置弹窗，显示：

- 渠道名称和分组名称。
- 当前上游倍率。
- 最大允许倍率输入；留空提示“不自动启停”。
- 计算比例输入，默认 `1`。
- 当前判定倍率和预计状态。
- 保存与关闭自动启停操作。

保存成功后立即刷新该渠道倍率数据并更新标签提示。未配置策略的标签保持现有倍率颜色，不增加误导性的启用标记。

同步分组编辑弹窗移除原“最大允许倍率”和“倍率计算比例”输入，避免存在两套冲突入口。

## Sub2API 聚合分组过滤

Sub2API 聚合页面只返回实际配置了智能调度池的分组：主调度池或保底池至少有一个账号或虚拟池条目。两个池均为空的远端分组不展示。

智能调度暂时关闭但池配置仍存在的分组继续展示，便于查看和维护现有配置。过滤在后端聚合 DTO 构造阶段执行，摘要和前端 Tab 数量都使用过滤后的结果。

## 错误与并发处理

- 策略保存期间禁用保存按钮，失败时保留输入并显示错误。
- 倍率扫描失败不执行该渠道策略，避免使用不完整数据。
- 策略协调按目标账号去重，防止同一轮重复请求。
- 同一账号意外匹配多个有效策略时，以“任一策略超过阈值即禁用”为准；全部策略均在阈值内才启用。
- 远端更新失败写入同步日志和服务日志，但不得记录 Admin Key、账号凭据或其他敏感字段。

## 测试范围

### 存储与 API

- 渠道分组策略创建、更新、删除和渠道删除级联清理。
- 有远端 ID 和仅名称两种分组身份匹配。
- 最大倍率、计算比例和缺失分组参数校验。
- 倍率响应正确合并策略与当前判定状态。

### 调度

- 超过阈值禁用，等于或低于阈值启用。
- 未配置、倍率缺失、分组消失时跳过。
- 同一渠道分组的多个托管账号全部更新。
- 禁用的同步账号、同步分组或目标不处理。
- 多策略命中同一账号时使用安全聚合决策。
- 定时扫描和手动刷新均触发协调。

### 页面

- 点击渠道分组标签可以打开并回显策略。
- 留空最大倍率可以关闭自动启停。
- 小数阈值和小数计算比例正确展示、保存和计算。
- Sub2API 聚合页面不展示主池、保底池均为空的分组。
