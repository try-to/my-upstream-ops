# 上游倍率自动控制 Sub2API 账号调度设计

## 目标

在 UpstreamOps 现有“上游动态同步”能力中，根据源渠道分组的上游原始倍率，自动启用或禁用目标 Sub2API 中对应托管账号的调度状态。

本功能只修改当前 UpstreamOps 项目，通过目标 Sub2API 已有 Admin API 设置账号 `schedulable`，不修改 Sub2API 源码，也不修改目标账号的 `active` / `inactive` 状态。

## 已确认规则

- 配置粒度为同步分组，同一同步分组下所有同步账号共用配置。
- 判定公式为：`判定倍率 = 上游原始倍率 * 计算比例`。
- 计算比例默认值为 `1`，允许大于 `0` 的小数。
- 最大允许倍率留空时，该同步分组不执行倍率自动启停。
- 判定倍率大于最大允许倍率时，设置目标账号 `schedulable=false`。
- 判定倍率小于或等于最大允许倍率时，设置目标账号 `schedulable=true`。
- 上游倍率恢复后直接启用调度。该最终决策允许覆盖同一轮模型测试失败产生的 `schedulable=false`。
- 上游倍率获取失败、绑定分组消失或没有可用倍率数据时，不执行倍率启停，不主动改变远端调度状态。
- 只处理 UpstreamOps 创建并记录映射的托管账号，不扫描或修改非托管账号。

## 方案选择

倍率控制接入现有 `ApplySyncGroup` 流程，不新增独立定时任务。

现有倍率扫描完成后已经调用 `SyncAllOnRateScan`，后者会重新应用所有启用的同步分组。手动应用同步分组也走同一条链路。因此，将倍率调度决策放在单账号应用流程末尾，可以同时覆盖定时扫描和手动应用，并避免两个任务并发修改同一个远端账号。

不采用独立倍率守卫任务，也不从监控模块直接修改远端账号。这样可以继续由同步模块独占目标账号管理职责。

## 数据模型

在 `UpstreamSyncGroup` 增加两个字段：

```text
rate_auto_toggle_threshold: nullable number
rate_auto_toggle_ratio: number, default 1
```

- `rate_auto_toggle_threshold` 为最大允许倍率。数据库值为 `NULL` 时表示关闭倍率自动启停，不额外增加布尔开关。
- `rate_auto_toggle_ratio` 为计算比例。新记录和缺失历史值统一按 `1` 处理。
- 最大允许倍率允许 `0` 及正数；计算比例必须大于 `0`。
- API 层拒绝 `NaN`、无穷值、负阈值和非正计算比例。
- 继续使用项目现有 GORM 自动迁移机制，不增加独立迁移框架。

DTO、创建输入、更新输入和前端类型使用同名字段。最大允许倍率在 JSON 中允许 `null`，计算比例始终返回归一化后的数值。

## 后端数据流

### 配置保存

1. 创建或更新同步分组时接收最大允许倍率和计算比例。
2. 空最大允许倍率保存为 `NULL`。
3. 未提交计算比例时保存为 `1`；显式提交非法值时返回参数错误。
4. 查询同步分组时回显两个字段。

### 调度决策

为单账号应用流程增加一个无副作用的倍率决策函数，输入为：

- 同步分组倍率配置；
- 同步账号绑定的 `SourceGroupID` / `SourceGroupName`；
- 本轮从源渠道读取的 `APIKeyGroup` 列表。

决策结果分为三种：

```text
enable   判定倍率 <= 最大允许倍率
disable  判定倍率 > 最大允许倍率
skip     未配置阈值或没有可靠的上游原始倍率
```

匹配源分组时优先使用 `SourceGroupID`，无法按 ID 匹配时再使用不区分大小写的 `SourceGroupName`。计算只使用匹配分组的原始 `Ratio`，不复用现有倍率换算模式，也不使用写入目标账号的 `rate_multiplier`。

### 应用顺序

1. 按现有逻辑创建或更新目标托管账号。
2. 按现有逻辑同步模型、执行账号测试及处理其他同步结果。
3. 在本轮单账号应用的最后计算倍率调度决策。
4. `enable` 调用目标 Sub2API Admin API 设置 `schedulable=true`。
5. `disable` 调用目标 Sub2API Admin API 设置 `schedulable=false`。
6. `skip` 不调用调度状态接口。

倍率决策放在模型测试之后，确保已确认的“倍率恢复直接启用调度”具有最终优先级。该逻辑不修改远端账号 `status` 字段；若账号本身为 `inactive`，本功能只保证其 `schedulable` 值被设置，不负责改变账号状态。

同步账号在本地显式设为禁用、源渠道被删除或源分组未绑定时，继续执行现有占位及禁用逻辑，不进入倍率恢复分支。倍率自动启停只作用于能够完成正常账号应用并取得可靠分组倍率的账号。

## 日志与通知

只有远端调度状态发生实际变化时，才向现有 `changes` 集合追加变化描述：

```text
账号N：上游原始倍率 X * 计算比例 Y = Z，超过最大允许倍率 T，已禁用调度
账号N：上游原始倍率 X * 计算比例 Y = Z，不超过最大允许倍率 T，已启用调度
```

沿用现有同步日志和 `upstream_sync_group_changed` 通知，不增加新的通知事件类型。重复应用且状态未变化时不产生变化通知，避免定时扫描刷屏。

设置远端调度状态失败属于当前同步账号应用失败，记录错误但不影响同一同步分组中其他账号继续处理。

## 前端交互

在“上游动态同步”的同步分组表单中增加：

- `最大允许倍率` 数字输入框：允许留空；留空提示“不自动启停调度”。
- `计算比例` 数字输入框：默认 `1`，支持小数，必须大于 `0`。

表单在最大允许倍率有值时显示只读公式提示：

```text
判定倍率 = 上游原始倍率 * 计算比例
```

编辑已有同步分组时正确回显配置。已有同步分组升级后阈值为空，因此默认不会改变任何目标账号的调度状态。

## 安全与兼容

- 目标 Sub2API Admin API Key 继续使用现有加密存储和解密调用链，不新增敏感字段。
- 日志只记录倍率、阈值和账号位置，不记录 API Key、Token、密码或目标管理密钥。
- 不修改 Sub2API 源码、数据库或部署配置。
- 不改变已有倍率换算、账号优先级排序、模型同步和远端账号状态管理逻辑。
- 未配置最大允许倍率的同步分组与升级前行为一致。

## 测试范围

### 后端

- 阈值为空时返回 `skip`，不调用远端调度接口。
- 计算比例默认 `1`，小数比例计算正确。
- 判定倍率大于阈值时禁用调度。
- 判定倍率等于或小于阈值时启用调度。
- 按源分组 ID 和名称均能找到原始倍率。
- 倍率读取失败、分组不存在或倍率数据缺失时保持远端状态。
- 模型测试失败后，倍率判定为 `enable` 时最终调度状态为启用。
- 倍率判定只修改 `schedulable`，不修改远端账号 `status`。
- 重复应用且调度状态未变化时不产生变化通知。
- 倍率扫描触发的 `SyncAllOnRateScan` 能应用新规则。
- 非法阈值和计算比例被拒绝。

### 前端

- 新建表单默认计算比例为 `1`。
- 阈值留空时提交 `null`。
- 小数比例可以保存并回显。
- 非正比例和负阈值不能提交。
- 编辑同步分组时正确回显阈值与比例。
