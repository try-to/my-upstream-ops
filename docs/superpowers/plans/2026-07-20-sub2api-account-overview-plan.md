# Sub2API 账号聚合页面实现计划

## 任务 1：扩展 AdminClient 全量账号读取

- 在 `backend/connector/sub2api/admin.go` 增加全量分页读取方法。
- 使用固定页大小逐页读取，遇到空页或不足一页时停止，并设置最大页数。
- 在 `backend/connector/sub2api/admin_test.go` 覆盖多页、末页和异常分页响应。

## 任务 2：实现服务层聚合与调度更新

- 在 `backend/syncer` 增加聚合 DTO，DTO 只包含页面展示需要的脱敏字段。
- 读取唯一目标并复用现有密钥解密链路。
- 拉取账号、分组、代理，关联同步分组和托管账号映射。
- 计算账号摘要、分组汇总、未分组汇总和托管来源。
- 增加单账号 `schedulable` 更新方法，响应只返回账号 ID 和新状态。
- 增加服务测试，覆盖聚合、托管识别、脱敏、唯一目标校验和调度更新。

## 任务 3：注册 HTTP API

- 在 `backend/api/upstream_sync.go` 注册聚合查询和调度更新路由。
- 严格校验账号 ID 和布尔请求体。
- 复用现有错误响应格式。
- 增加 API handler 测试覆盖非法输入和成功响应。

## 任务 4：实现前端页面

- 在 `frontend/lib/api-types.ts` 增加聚合 DTO 类型。
- 新增 `frontend/app/sub2api-overview-page.tsx`。
- 实现加载、空数据、保留旧数据的刷新失败状态。
- 实现摘要指标、分组汇总、账号明细、搜索和组合筛选。
- 实现单账号调度开关，请求期间锁定，失败时恢复。
- 使用现有 UI 组件和响应式表格规范。

## 任务 5：接入导航

- 在 `frontend/src/main.tsx` 注册 `/sub2api-overview` 路由。
- 在顶部导航增加图标入口和工具提示。
- 保持移动端按钮尺寸稳定，不挤压标题和其他操作。

## 任务 6：验证与交付

- 运行 Go 格式化和 `go test ./...`。
- 运行前端 ESLint、TypeScript 检查和生产构建。
- 启动本地服务，使用桌面和移动端视口验证页面布局及交互。
- 检查页面请求响应不包含凭据和管理密钥。
- 执行常见硬编码密钥、Token、密码和私钥特征扫描。
- 提交代码并推送到用户仓库的 `origin/main`。
