# Sub2API 智能调度聚合实现计划

## 任务 1：按智能调度路由聚合

- 全量读取 Sub2API 分组、账号和代理。
- 以 `model_routing` 的主池、保底池为分组账号池来源。
- 正 ID 关联真实上游账号；负 ID 聚合为 OAuth 虚拟池计数。
- 聚合响应不返回 OAuth 授权账号明细和凭据。

## 任务 2：调度操作

- 保留真实上游账号 `schedulable` 开关。
- 新增分组智能调度权重更新接口。
- 权重限制为 `1-999` 整数。
- 保存前校验池 ID 数量和顺序，禁止通过权重接口修改池结构。
- 只替换两个权重键，保留模型专用路由和分组其他配置。

## 任务 3：分组 Tab 页面

- 使用横向可滚动 Tab 切换分组，一次只显示一个分组。
- 在主调度池和保底池条目中提供权重输入。
- 当前分组有改动时启用整组保存按钮。
- 保留搜索、平台和智能调度状态筛选。
- 验证桌面与移动端无页面级横向溢出。

## 任务 4：测试与交付

- 覆盖聚合脱敏、虚拟池计数、路由字段更新、模型路由保留和非法输入。
- 运行 `go test ./...`。
- 运行前端 ESLint、TypeScript 检查和生产构建。
- 执行敏感信息扫描。
- 提交并只推送到用户仓库 `origin/main`。
