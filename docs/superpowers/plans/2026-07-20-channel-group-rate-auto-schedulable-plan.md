# 渠道分组倍率自动调度实施计划

## 任务 1：策略存储

- 新增独立渠道分组倍率策略表。
- 使用渠道 ID 与稳定分组键唯一定位配置。
- 支持最大倍率、计算比例、分组 ID 与名称回退。
- 渠道删除时级联清理策略，倍率快照删除时保留策略。

## 任务 2：策略 API

- 在渠道倍率响应中合并最大倍率、计算比例、判定倍率和预计调度状态。
- 新增渠道分组策略保存与关闭接口。
- 校验渠道、当前倍率分组及数值范围。
- 策略保存后立即尝试协调对应托管账号。

## 任务 3：调度协调器

- 根据同步账号的源渠道和源分组找到渠道分组策略。
- 只处理启用的同步账号、同步分组和同步目标。
- 通过托管映射定位 Sub2API 真实账号。
- 超过最大倍率禁用调度，恢复后启用调度。
- 定时倍率扫描、手动单渠道同步、批量同步和手动应用同步分组后执行协调。
- 保留原有动态同步分组应用流程，并保证倍率策略最后执行。

## 任务 4：前端配置

- 渠道卡片分组倍率标签改为可点击配置。
- 弹窗提供最大倍率、计算比例、实时判定和关闭策略操作。
- 已配置标签直接显示最大倍率和预计启停状态。
- 移除同步分组编辑中的旧倍率阈值入口。

## 任务 5：Sub2API 聚合过滤

- 后端过滤主调度池和保底池均为空的远端分组。
- 保留池配置存在但智能调度暂时关闭的分组。
- 汇总数量和前端 Tab 使用过滤后的分组集合。

## 任务 6：验证与交付

- 覆盖策略存储、API 校验、名称回退、多账号启停和空分组过滤。
- 运行完整 Go 测试。
- 运行前端 ESLint、TypeScript 检查和生产构建。
- 执行敏感信息扫描，提交并只推送到 `origin/main`。
